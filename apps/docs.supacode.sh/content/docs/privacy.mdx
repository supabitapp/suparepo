title: Privacy
description: How Supacode handles your data
---

Supacode is a local-first application. Your code, terminal sessions, and git history never leave your machine. This page explains what data Supacode collects, where it goes, and how to turn it off.

## What stays on your machine

Everything that matters:

- **Terminal I/O** — All keystrokes, command output, and agent conversations stay local
- **Code and git data** — Supacode reads your repositories to manage worktrees. None of this is transmitted anywhere
- **Settings** — Stored at `~/.supacode/settings.json` on disk
- **Worktrees** — Managed under `~/.supacode/repos/` as standard git worktrees

Supacode has no backend. There are no user accounts, no cloud sync, and no server-side storage.

## Analytics

When enabled, Supacode sends anonymous usage events to [PostHog](https://posthog.com) to help us understand which features are used and how. Events include things like `worktree_created`, `terminal_tab_opened`, and `settings_changed`.

These events carry **no contextual data** — no file paths, repository names, branch names, code content, or environment variables. The only identifier is your device's hardware UUID.

## Crash reports

When enabled, Supacode sends crash and error data to [Sentry](https://sentry.io). This includes operation types and error codes when something goes wrong (for example, a failed git operation). It does not include error messages, stack traces from your code, or any repository content.

## GitHub integration

Supacode delegates all GitHub operations to the `gh` CLI installed on your machine. Supacode never stores GitHub credentials or makes direct API calls — it invokes `gh` as a subprocess and reads the output. Your GitHub authentication is managed entirely by the GitHub CLI.

## Opting out

All telemetry is opt-in via Settings (**⌘,**):

- **Share analytics with Supacode** — Toggle off to stop PostHog events
- **Share crash reports with Supacode** — Toggle off to stop Sentry reports
- **GitHub integration** — Toggle off to disable all `gh` CLI calls
  Changes to analytics and crash reporting take effect after restarting the app.

## Third-party services

| Service                        | Purpose         | What it receives             |
| ------------------------------ | --------------- | ---------------------------- |
| [PostHog](https://posthog.com) | Usage analytics | Device UUID, event names     |
| [Sentry](https://sentry.io)    | Error tracking  | Operation types, error codes |

No session recordings, screen captures, or user data. Ever.
